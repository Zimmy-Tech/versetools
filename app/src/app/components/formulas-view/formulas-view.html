Power System

Every ship has a fixed power budget generated by its power plant(s). Components draw from this budget via allocatable power pips. The power bar UI lets you redistribute pips within the total output.

Total Power Output

Total Power Output = sum( Plant Output )  for all equipped power plants

Power plants are always-on components with no pip control. Their output sets the ceiling for all other allocations.

Total Power Used

Total Power Used = sum( Component Pips ) + Weapon Pips + Tool Pips + Tractor Pips

Cannot exceed Total Power Output. The UI clamps allocation changes to keep usage within budget.

Weapon Power Pool

Max Weapon Pips = min( Ship Pool Size, ceil( Total Weapon Power Cost ) )

where Total Weapon Power Cost = sum( Weapon Power Cost )  for all equipped weapons

The weapon pool is a fixed-size pool declared per-ship. Maximum usable pips are capped by whichever is smaller: the pool size or the total power cost of equipped weapons (rounded up).

Power Band Modifier

Band Modifier at Pips:
  if Pips <= 0  →  0  (component off)
  if no bands   →  1  (full efficiency)
  otherwise     →  efficiency of highest band whose threshold <= current Pips

Many components (shields, coolers, QDs) have tiered power bands. The modifier scales the component's effectiveness at the current pip level. Each band defines a pip threshold and the efficiency multiplier that applies above it.

Minimum Pip Block (Band-Gap)

Min Block = Band 2 Threshold - Band 1 Threshold    (if 2+ bands)
Min Block = 1                                       (if single band)

The minimum allocation for a component is derived from the gap between its first two power bands. This prevents the component from operating below its lowest effective threshold.

Default Power Allocation

Weapons 50% of pool size Shields 50% of max, at least min threshold Coolers minimum band pip Life Support 1 pip (always on) Radar max(1, round(Power Cost × Idle Consumption Fraction)) Quantum Drive 0 pips (off at spawn) Tools 1 pip per tool (always on) Tractor 2-pip block toggle

Weapons & DPS

Weapon damage calculations depend on weapon type (ballistic vs energy), fire rate, ammo mechanics, and heat management. DPS is computed over a 10-second window to capture burst/regen cycling.

Energy Weapon Ammo

Effective Magazine  = Base Magazine × Ship Ammo Multiplier
Magazine Cap        = floor( Effective Magazine )
Ammo at Pips(N)     = min( round( N × Effective Magazine / Total Weapon Power ), Magazine Cap )

where:
  Base Magazine        = weapon's default ammo capacity (typically 75 rounds)
  Ship Ammo Multiplier = per-ship engineering buff (ranges 1.75 – 6.0)
  Total Weapon Power   = sum of power cost across all weapons in the pool
  N                    = currently allocated weapon pips

Energy weapons share ammo from a pooled budget. Each weapon's share is proportional to its power cost relative to the total. More pips = more ammo per weapon.

Ballistic Weapon Burst

Burst Rounds = floor( Heat Capacity / Heat Per Shot )
Burst Time   = Burst Rounds / ( Cyclic Rate / 60 )

Ballistic weapons have fixed magazine capacity and fire until overheating. The burst window is determined by how fast heat accumulates versus the weapon's thermal ceiling.

Alpha Damage

Alpha Damage = Physical + Energy + Distortion + Thermal

Sum of all damage types dealt per shot. Each projectile carries a breakdown of damage across four categories, extracted from the weapon's linked ammo record.

Sustained DPS (10-second window)

Ballistic:
  Cycle Time  = Burst Time + Overheat Cooldown
  Full Cycles = floor( 10 / Cycle Time )
  Remaining   = 10 - Full Cycles × Cycle Time
  Total Dmg   = Full Cycles × Burst Dmg + min(Remaining, Burst Time) × Rounds/Sec × Alpha

Energy:
  Regen Time  = Regen Delay + Base Magazine / Regen Rate
  Cycle Time  = Burst Time + Regen Time
  (same cycle calculation as ballistic)

Computes total damage across burst/reload cycles within a 10-second window. This captures the real sustained output including downtime. "Regen Delay" is the pause before ammo begins regenerating; "Regen Rate" is ammo recovered per second.

Energy Weapon Sustained Ratio

Sustained Ratio(N) = (300 × N) / (300 × N + Cyclic Rate RPM)

where:
  300 = Regen Constant = (Magazine / Restock Cycles) × Regen Rate × 60
      = (75 / 3) × 15 = 300
  N   = weapon bars allocated (capped at max restock cycles, typically 3)

Fraction of theoretical max DPS achievable when sustaining fire continuously. The regen constant represents how quickly a weapon can replenish ammo between bursts. Higher ratio = less downtime.

Missile Alpha

Missile Alpha = sum( Per-Missile Damage )  for all equipped missiles

Total one-volley damage from all missile racks. Uses the same per-projectile damage extraction as guns.

Shields

Shield mechanics use a primary/reserve model. The first two equipped shields are "active" (regen + HP + power pips). Additional shields beyond the second are "reserve" (HP pool only, no regen, auto-activate).

Total Shield HP

Total Shield HP = sum( Shield Health )  for ALL equipped shields (primary + reserve)

Shield Regen Rate (Linear Model)

Shield Regen Rate = Max Regen × ( Allocated Pips / Max Pips )

where:
  Max Regen      = sum( Shield Regen Speed )   for primary shields only (first 2)
  Allocated Pips = sum( Current Pips )         for primary shields only
  Max Pips       = sum( max(1, Power Ceiling - 1) )  for primary shields only

Regen scales linearly with allocated pips. Only the first two shields contribute to regeneration. Reserve shields (3rd+) add HP but never regen.

Shield Regen Time

Regen Time = Total Shield HP / Shield Regen Rate

Time to regenerate the full shield pool from zero. Practical time is longer due to damage-taken delays and downed-state recovery pauses.

Shield Power Distribution

Phase 1: Give each shield its minimum (band-gap) first
Phase 2: Distribute remaining pips up to each shield's max

Min Pips = Band 2 Threshold - Band 1 Threshold  (per shield)
Max Pips = Power Ceiling - 1                     (per shield)

The combined shield bar distributes pips across all primary shields, ensuring each meets its minimum threshold before allocating extras.

Resistances & Absorptions

Resistance range: [Min, Max] per damage type
  Physical, Energy, Distortion, Thermal, Biochemical, Stun

Positive = reduces damage, Negative = amplifies damage
Absorption: fraction of damage applied to shield vs hull

Each shield has min/max resistance and absorption values per damage type. These are static properties, not affected by power pips.

Cooling

Coolers generate coolant that other components consume. If cooling demand exceeds supply, components overheat. The cooling gauge shows the supply/demand ratio as a percentage.

Cooling Supply (per cooler)

Cooler Supply = Base Cooling Output × Band Modifier at Current Pips

if Pips = 0  →  Supply = 0

Each cooler produces coolant scaled by its power band modifier at the current pip allocation. Total supply is the sum across all equipped coolers.

Cooling Demand (per component)

Power Plant:              Demand = max(0, Plant Output - 1)    [always on]
Radar / Quantum Drive:    Demand = Allocated Pips              [linear with pips]
Shield / Cooler / Life Support:
                          Demand = Power Cost × Band Modifier at Current Pips
Tools:                    Demand = 1 per tool                  [always on]

Different component types compute demand differently. Power plants have fixed demand based on output. Most others scale with their band modifier.

Cooling Percentage

Cooling % = round( Total Cooling Demand / Total Cooling Supply × 100 )

Displayed on the cooling gauge. Values above 100% indicate thermal overload — components will begin overheating.

Flight & Thrusters

Flight stats are both crowd sourced and calculated. Rotation rates scale with thruster power pips between base and boosted values.

Rotation Rate Scaling

Thruster Multiplier = (Current Thruster Pips - 1) / (Max Thruster Pips - 1)
  (if Max Thruster Pips <= 1: Thruster Multiplier = 1)

Rotation Rate = Base Rate + Thruster Multiplier × (Boosted Rate - Base Rate)

Rotation rates (pitch, yaw, roll) interpolate linearly between their base value (at 1 pip) and their boosted value (at max pips). The thruster power bar controls this scaling.

Speed Stats

SCM Speed         —  Standard Combat Maneuvering speed
NAV Speed         —  Navigation (cruise) speed
Boost Speed (Fwd) —  Afterburner forward speed
Boost Speed (Bwd) —  Afterburner reverse speed

These are direct extractions from the flight controller, not computed values.

Acceleration

Directional: Forward, Retro, Strafe, Up, Down
Afterburner: AB Forward, AB Retro, AB Strafe, AB Up, AB Down

Acceleration values are community-tested where available. The theoretical formula requires 3D thruster position vectors and orientation quaternions that are not fully extractable from game files. Afterburner multipliers are confirmed from the flight controller's afterburner definition.

} @case ('mining') {

Mining

Mining lasers have base stats that are modified by equipped mining modules. Laser modifiers are multiplicative; module modifiers are additive.

Mining Power with Modules

Combined Power Mult = product( Module Power Multiplier )  for all equipped modules

Combined Min Power = round( Laser Min Power × Combined Power Mult )
Combined Max Power = round( Laser Max Power × Combined Power Mult )

Power multipliers from modules stack multiplicatively. Min and max laser power are both scaled by the combined multiplier.

Mining Stat Modifiers

For each stat (Instability, Optimal Window, Optimal Rate,
                Resistance, Shatter Damage, Inert Materials, Overcharge):

  Total Stat = Laser Base Value + sum( Module Adjustment )  for all equipped modules

Module stats stack additively onto the laser's base values. Values are percentage adjustments (e.g., -35 = -35% instability reduction).

} @case ('radar') {

Radar

Radar lock range scales linearly with power allocation between minimum and maximum aim distances.

Lock Range

Pip Fraction = min( Current Pips / Max Pips, 1 )
Lock Range   = round( Min Lock Distance + (Max Lock Distance - Min Lock Distance) × Pip Fraction )

where Max Pips = radar's total power cost

At minimum power, lock range equals the radar's shortest lock distance. At full power, it reaches maximum lock distance. The scaling is linear between these bounds.

Radar Minimum Power

Radar Min Pips = max( 1, round( Power Cost × Idle Consumption Fraction ) )

The minimum allocation is derived from the radar's idle consumption fraction (typically 0.25). This is the lowest power level at which the radar operates.

Cargo

Cargo capacity is calculated from the interior dimensions of each cargo grid in the ship's default loadout.

SCU Capacity

SCU Per Grid = floor(Width / 1.25) × floor(Length / 1.25) × floor(Height / 1.25)
Total Cargo  = sum( SCU Per Grid )  for all cargo grids in default loadout

Each axis is divided by the SCU unit size (1.25m) and floored. The product gives the number of 1-SCU boxes that fit in that grid.

Crafting

The crafting system uses quality-based modifiers on ingredients to determine the final stats of crafted items. Blueprints are unlocked through mission rewards.

Quality Modifier Interpolation

t        = (Clamped Quality - Range Start) / (Range End - Range Start)
Modifier = Start Multiplier + t × (End Multiplier - Start Multiplier)

Clamped Quality = clamp( Quality, Range Start, Range End )
Quality range: 0 – 1000

Each ingredient has quality modifiers that map a quality value (0–1000) to a stat multiplier via linear interpolation between the curve's start and end points.

Combined Modifier (Multi-Ingredient)

Combined Modifier = product( Ingredient Modifier )  per stat

Each ingredient's modifier for a given stat multiplies together.

When multiple ingredients affect the same stat, their modifiers are multiplicative. A modifier of 1.0 is neutral, >1.0 is a buff, <1.0 is a nerf.

Dismantle Returns

Returned Quantity = floor( Original Quantity × 0.5 )

Resources < 0.01 SCU return nothing.
Sub-unit items: 3 → 1, 1 → 0 (floor of half)

Dismantling returns 50% of crafting inputs, rounded down. Very small quantities may return nothing.

Blueprint Sources

Blueprints are unlocked as mission rewards.
The Crafting tab cross-references mission data to show which
missions award each blueprint.

Chain prerequisites may apply — some blueprints are only available after completing earlier missions in a sequence.

Missions

Mission Data is crowd sourced, and is a Work in Progress.

Contract Cooldowns

Completion Cooldown = time before the contract can be retaken (minutes)
Abandon Cooldown    = time before an abandoned contract reappears (minutes)

Each contract has two independent cooldown timers. The completion cooldown applies after successfully finishing the mission; the abandon cooldown applies if the player drops it.

Chain Prerequisites

A mission may require completing specific prior missions.
Work in Progress.

Chain missions require completing earlier missions in the sequence. Prerequisite identifiers are mapped to display names during extraction.

Reward Calculation

Work in Progress.
Known inputs: base reward, difficulty tier, reputation scope.
Suspected modifiers: standing level, risk factor, location.

The exact reward formula has not been fully decoded. Displayed reward values are currently extracted as static values where available.

Standing Requirements

Standing tiers: Applicant, Rank I – VI
Work in Progress.

Reputation/standing gates are defined per-contract but the numeric thresholds for each tier are not yet fully extracted.
